# Tags Taxonomy for Simulation Technologies Documentation

## Purpose
This document defines the standardized tags taxonomy for organizing simulation technologies documentation (Gazebo, Unity, sensors).

## Tag Categories

### Gazebo-Specific Tags
- `gazebo`: General Gazebo concepts
- `physics`: Physics simulation concepts
- `simulation`: General simulation concepts
- `models`: Model creation and management
- `worlds`: World/environment creation
- `plugins`: Plugin development and usage
- `urdf`: URDF integration with Gazebo
- `ros-integration`: ROS integration with Gazebo

### Unity-Specific Tags
- `unity`: General Unity concepts
- `digital-twin`: Digital twin concepts
- `3d-visualization`: 3D visualization techniques
- `hri`: Human-robot interaction
- `ui`: User interface development
- `assets`: Asset management
- `rendering`: Rendering techniques
- `xr`: Extended reality (AR/VR) applications

### Sensor Simulation Tags
- `sensors`: General sensor concepts
- `lidar`: LiDAR simulation
- `cameras`: Camera simulation (depth, RGB, etc.)
- `imu`: IMU simulation
- `sensor-fusion`: Sensor fusion techniques
- `perception`: Perception system development
- `validation`: Validation and testing

### Simulation Concepts
- `humanoid`: Humanoid robotics applications
- `robotics`: General robotics concepts
- `simulation`: General simulation concepts
- `environments`: Environment creation
- `integration`: System integration
- `validation`: Validation and verification

### Educational Levels
- `beginner`: Content for beginners
- `intermediate`: Content for intermediate learners
- `advanced`: Advanced concepts
- `tutorial`: Step-by-step tutorials
- `reference`: Reference material

## Tagging Guidelines

1. Use 3-5 tags per document
2. Always include at least one technology-specific tag (e.g., `gazebo`, `unity`, `sensors`)
3. Include an educational level tag when appropriate
4. Use specific tags rather than general ones when possible
5. Combine domain-specific tags with technology tags (e.g., `gazebo` + `humanoid`)
